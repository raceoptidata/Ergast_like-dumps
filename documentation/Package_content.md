## Overview

This package contains a full extraction of Formula 1 data sourced from **Jolpica**, 
processed and transformed into a structure compatible with the **Ergast** database. 
It is intended for use in data analysis, performance testing, or academic purposes.

---

## Package Contents

### 1. Data

- **`.csv` files**: Raw exports of all tables in text format.
- **MySQL dump**: Native SQL export of the working database (via `mysqldump`).
- **PostgreSQL dump**: Generated after importing `.csv` files into a clean PostgreSQL instance.
- **`.bacpac` file**: SQL Server export produced through Azure Data Factory.

### 2. Documentation

- `README.md`: This document.
- `Database scheme.txt`: Structure and schema description of the Ergast database (author: Chris Newell).
- `Ergast_MCD.JPG`: Conceptual data model diagram.
- `ErgastDump_HowTo.pdf`: Guideline, how to import the dumps
- `manifest_*.json`: Detailed execution log of the pipeline (steps, durations, statuses, etc.).

---

## Pipeline Steps

1. **Data Collection from Jolpica**
   - Calls to `https://api.jolpi.ca/...` endpoints
   - Normalization, data cleaning, and transformation
   - Joins with reference tables (races, drivers, constructors, etc.)
   - Load into the MySQL working database

2. **MySQL Transfer**
   - SQL-based transfer from working database to target database
   - Executed from SQL template as a single transaction

3. **Exports**
   - SQL export from MySQL using `mysqldump`
   - `.csv` exports of all database tables
   - `.bacpac` export of SQL Server database via Azure Data Factory
   - PostgreSQL dump generated by loading `.csv` files and exporting the database

4. **Packaging**
   - Creation of a `.zip` archive containing all generated files
   - Documentation and manifest are automatically included
